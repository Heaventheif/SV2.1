# 🤖 SunkenBot v2.1 — بوت فيسبوك ماسنجر

> بوت ماسنجر مكتوب بـ Node.js يعمل على حسابك كـ Userbot داخل مجموعات فيسبوك، مع دعم الذكاء الاصطناعي، تحميل الميديا، الشطرنج، القرآن الكريم، وأكثر.

---

## 🆕 تحديثات هذه النسخة (حماية الحساب + الذاكرة + الأداء)

بعد مراجعة شاملة للكود، أُضيفت التحسينات التالية:

**حماية الحساب من الحظر**
- طابور الإرسال (`safeSend`) أصبح **لكل مجموعة (threadID) على حدة** بدل طابور عام واحد — لا تتأخر ردود مجموعة بسبب ازدحام مجموعة أخرى، مع بقاء فاصل ثانية واحدة محفوظاً داخل نفس المجموعة.
- أي استدعاء `api.sendMessage` داخل أي أمر يمر **تلقائياً** عبر `safeSend` الآن (عبر تغليف `api` في `index.js`)، حتى لو كان الأمر يستدعي `api.sendMessage` مباشرة دون تعديل كل ملف يدوياً.
- أمر `.adduser`: رُفع الـ cooldown من 5 إلى 45 ثانية، وأُضيف **حد أقصى 8 إضافات/يوم لكل مشرف** (في الذاكرة)، لأن إضافة الأعضاء هي أكبر سبب لحظر/تقييد حسابات فيسبوك.
- أُزيل أي مفتاح API افتراضي مكتوب صراحة في الكود (`FB_GRAPH_ACCESS_TOKEN` في `adduser.js`) — يجب ضبطه من البيئة فقط.
- أُضيف **rate limiting عام لكل مستخدم** (5 أوامر كحد أقصى كل 10 ثوانٍ) فوق الـ cooldown الخاص بكل أمر، لمنع إغراق طابور الإرسال بتبديل سريع بين أوامر مختلفة.

**الذاكرة وقواعد البيانات**
- `global.usersData` و`global.globalData` أصبحا مرتبطين فعلياً بـ MongoDB: قراءة كسولة (lazy-load) عند أول استخدام، وكتابة دورية مجمّعة (batch) كل 5 دقائق بدل الكتابة عند كل تفاعل — بياناتك لم تعد تُفقد عند إعادة التشغيل (إن وُجد `MONGO_URI`).
- أُضيف مخطط `GlobalDataModel` جديد في `db/schemas.js` لتخزين البيانات العامة.
- `global.client.reactionListener` أصبح يُنظَّف تلقائياً ضمن دورة التنظيف العامة كل 30 دقيقة (عبر Proxy يسجّل وقت كل إدخال)، بدل الاعتماد فقط على منطق حذف متفرق في كل ملف أمر.
- أُضيفت دالة تنظيف للملفات المؤقتة اليتيمة في `os.tmpdir()` (أقدم من ساعة)، تعمل عند الإقلاع وكل 30 دقيقة، لحماية مساحة التخزين من تراكم ملفات لم تُحذف بسبب أخطاء غير متوقعة.
- إغلاق سليم (graceful shutdown) عند `SIGTERM`/`SIGINT` يحفظ آخر التغييرات قبل إيقاف العملية (مفيد عند إعادة النشر على Render).

**الأداء والسرعة**
- أُضيف كاش بسيط بـ TTL (`utils/cache.js`) مُفعَّل حالياً على مسار البحث الداخلي `/yt/search` (5 دقائق) لتقليل زمن الاستجابة عند تكرار نفس البحث وتوفير حصة الـ APIs الخارجية. يمكن تطبيق نفس النمط على أوامر أخرى (ترجمة، حقائق...).
- أُزيل `await` غير ضروري على معالجات الأحداث (fire-and-forget أصلاً) داخل حلقة استقبال الرسائل (`listenMqtt`) لتفادي أي تأخير تتابعي بين الأحداث الواردة.
- نقطة `/health` الآن تعرض أيضاً حالة طوابير الإرسال (`sendQueue`) للمراقبة.

**أخرى**
- أُضيف ملف `.env.example` يوضح كل متغيرات البيئة المطلوبة/الاختيارية.
- تحقق مبكر عند الإقلاع يحذّر بوضوح إن كانت بيانات الدخول أو `MONGO_URI` غير مضبوطة، بدل اكتشاف الغياب لاحقاً أثناء التشغيل.

> ⚠️ تذكير: المكتبة المستخدمة (`@dongdev/fca-unofficial`) تُسجّل دخولاً غير رسمي لحساب فيسبوك حقيقي، وهذا يخالف شروط استخدام فيسبوك بحد ذاته بصرف النظر عن جودة الكود — استخدم دائماً حساباً مخصصاً للبوت وليس حسابك الشخصي، وكن على وعي بأن الحظر يبقى احتمالاً وارداً.

## 🚀 خطوات التشغيل

### 1. تثبيت المكتبات

```bash
npm install
```

### 2. إنشاء ملف `.env`

انسخ `.env.example` إلى `.env` واملأ القيم الحقيقية:

```bash
cp .env.example .env
```

```env
# بيانات حساب فيسبوك البوت
FB_EMAIL=email@example.com
FB_PASSWORD=yourpassword

# إن كان الحساب يستخدم 2FA (اختياري)
FB_2FA_SECRET=

# قاعدة البيانات (موصى بها بشدة الآن — بدونها تُفقد بيانات
# المستخدمين عند كل إعادة تشغيل، راجع قسم "قاعدة البيانات" أدناه)
MONGO_URI=mongodb+srv://...

# مفاتيح API للأوامر الذكية (حسب ما تريد تفعيله)
CEREBRAS_API_KEY=
HF_SPACE_URL=
HF_SCRAPER_URL=

# اختياري — يُستخدم في .adduser لتحسين استخراج UID من اليوزرنيم
FB_GRAPH_ACCESS_TOKEN=
RAPIDAPI_KEY=
```

### 3. إعداد `config.json`

```json
{
  "admins": ["ID_حسابك_على_فيسبوك"],
  "moderators": [],
  "developers": [],
  "vips": [],
  "Prefix": ["."],
  "botName": "SunkenBot"
}
```

> **كيف أعرف ID حسابي؟** استخدم أمر `.uid` داخل المجموعة بعد تشغيل البوت.

### 4. تشغيل البوت

```bash
# تشغيل عادي
npm start

# تشغيل مع إعادة تحميل تلقائية عند التعديل
npm run dev
```

عند أول تشغيل ناجح ستشوف في الـ Console:
```
[SUCCESS] البوت متصل بنجاح
[INFO] تم تحميل XX أمر
[DB] ✅ MongoDB متصل بنجاح  ← (إن وضعت MONGO_URI)
```

## 💾 قاعدة البيانات (MongoDB) — موصى بها بشدة

البوت يشتغل **بدون** قاعدة بيانات، لكن إن ربطته بـ MongoDB يحصل على:

- **حفظ بيانات المستخدمين بشكل دائم** — XP، المستوى، الرصيد، عدد الرسائل (الآن مرتبطة فعلياً بـ `usersData`، تُقرأ كسولاً عند أول استخدام وتُكتب دفعة واحدة كل 5 دقائق)
- **بيانات عامة دائمة** (`globalData`) عبر `GlobalDataModel` الجديد
- **ذاكرة محادثة AI** — يتذكر سياق المحادثة مع Gemini/Cerebras
- **إعدادات المجموعات** — رسالة ترحيب، حماية ضد السبام، prefix مخصص

بدون MongoDB كل شيء يُحفظ في الذاكرة المؤقتة ويُمسح عند إعادة التشغيل (يظهر تحذير واضح عند الإقلاع إن كان `MONGO_URI` غير مضبوط).

## ➕ إضافة أمر جديد

أنشئ ملف `commands/mycommand.js` بهذا الهيكل:

```js
module.exports = {
  config: {
    name: "mycommand",       // اسم الأمر
    aliases: ["alias1"],     // أسماء بديلة
    role: 0,                 // 0 = للجميع، 1 = مشرفين فقط
    countDown: 5,            // cooldown بالثواني
    category: "أدوات",
    description: "وصف أمري الجديد",
  },

  onStart: async ({ api, event, args }) => {
    const { threadID, messageID } = event;
    // args = باقي الكلمات بعد الأمر
    // مثال: ".mycommand مرحبا" → args = ["مرحبا"]

    await api.sendMessage("مرحباً! الأمر شغال ✅", threadID, messageID);
  }
};
```

البوت يحمّل الأوامر تلقائياً من مجلد `commands/` عند كل تشغيل.

## 🔧 استكشاف الأخطاء

| المشكلة | الحل |
|---------|------|
| البوت لا يتصل بفيسبوك | تأكد من صحة `FB_EMAIL` و `FB_PASSWORD` في `.env` |
| أوامر AI لا تعمل | تأكد من وضع مفتاح API الصحيح في `.env` |
| MongoDB لا يتصل | تأكد من صحة `MONGO_URI` — البوت يكمل بدونها لكن بدون حفظ دائم |
| أمر لا يُحمَّل | تحقق من console عند التشغيل لرسائل `[WARN]` |
| `.adduser` يرفض التنفيذ برسالة "وصلت للحد الأقصى" | وصلت لحد 8 إضافات/يوم — هذا مقصود لحماية الحساب، انتظر حتى اليوم التالي |
| الردود بطيئة في مجموعة مزدحمة | راجع `/health` لمعرفة عدد الرسائل المنتظرة في `sendQueue` لتلك المجموعة |

## 📝 ملاحظات مهمة

- ⚠️ **لا ترفع ملف `.env`** على GitHub — فيه كلمات مرور ومفاتيح سرية
- استخدم حساباً فيسبوك **مخصصاً للبوت**، وليس حسابك الشخصي
- فيسبوك قد يحظر الحساب إن استخدم البوت بكثافة — هذا خارج عن إرادتنا، رغم أن هذه النسخة تضيف حماية إضافية (طوابير لكل مجموعة، حد يومي لـ adduser، rate limiting عام)
- الـ Prefix الافتراضي هو النقطة `.` — يمكن تغييره من `config.json`
- إن أردت ربط حد `.adduser` اليومي بقاعدة البيانات بدل الذاكرة (ليبقى محفوظاً عبر عمليات إعادة التشغيل)، عدّل `commands/adduser.js` ليستخدم `UserModel` بدل `_dailyAddLog` المحلي
